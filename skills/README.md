# YanCe Policy Agent - Skills (研策政策智能体 - 技能包)

面向园区企业服务场景的 AI 技能包集合，覆盖政策匹配、企业画像、材料生成、运营报告和政策问答五大核心场景。

## 技能包一览

| 技能 | 说明 | 触发词示例 |
|------|------|-----------|
| [policy-matcher](./policy-matcher/) | 8维度加权政策匹配引擎 | "帮我匹配政策"、"能申请什么政策" |
| [company-profiler](./company-profiler/) | 企业画像结构化录入与标签分层 | "帮我做企业画像"、"录入企业信息" |
| [brief-maker](./brief-maker/) | 材料清单与政策简报自动生成 | "生成材料清单"、"写一份政策简报" |
| [park-ops-report](./park-ops-report/) | 14章节园区服务运营报告 | "生成园区服务报告"、"园区运营周报" |
| [policy-copilot](./policy-copilot/) | 带引用溯源的政策问答 Agent | "政策问答"、"这个政策什么意思" |

## 技能协作流程

```
企业信息录入 ──> 企业画像 ──> 政策匹配 ──> 材料清单/简报
   (company-profiler)    (policy-matcher)    (brief-maker)
                                    │
                            政策问答 (policy-copilot) ←── 随时可查

园区服务数据 ──> 运营报告 (park-ops-report)
```

## 跨平台使用

### WorkBuddy (工作搭档)
- 适用场景：园区服务专员日常工作、企业客户对接
- 使用方式：在 WorkBuddy 中安装技能包，通过对话或表单触发
- 优势：支持文件导出（PDF/Excel）、工单系统集成

### CodeBuddy (代码搭档)
- 适用场景：开发政策服务平台、集成 API 接口
- 使用方式：参考 SKILL.md 中的 Schema 定义实现后端逻辑
- 优势：结构化 JSON/YAML 输出，便于系统集成

### 千问 / Qianwen (通义千问)
- 适用场景：自然语言对话式政策咨询、移动端快速查询
- 使用方式：将 SKILL.md 内容作为 System Prompt 或 Skill 配置导入
- 优势：多轮对话、追问细节、自然语言理解

## 安装与使用

### 在 QoderWork 中使用
将本 skills 目录放置在 QoderWork 可识别的技能目录下，技能将自动被发现和加载。

### 手动使用
1. 复制目标技能的 `SKILL.md` 文件
2. 将其内容作为 System Prompt 或技能配置导入到目标平台
3. 使用触发词或指令激活技能

## 通用原则

所有技能包遵循以下设计原则：

1. **非金融边界**: 所有技能明确声明不提供投资建议、融资方案或财务咨询
2. **引用溯源**: 政策相关内容必须附带来源、文号、日期等引用信息
3. **人工确认点**: 关键决策节点标注"建议人工复核"，不完全替代专业判断
4. **结构化输出**: 统一使用 YAML/JSON 格式输出，便于跨平台解析
5. **渐进式收集**: 缺失信息时主动追问，不猜测用户意图

## 贡献与改进

### 报告问题
如果发现技能逻辑错误、遗漏场景或输出格式问题，请在项目 Issue 中反馈。

### 贡献新技能
1. 创建新的技能目录：`skills/your-skill-name/SKILL.md`
2. 遵循 YAML frontmatter 格式（name、description、version）
3. 包含：触发条件、输入参数、执行步骤、输出格式、边界处理
4. 确保包含非金融声明
5. 提交 Pull Request

### 技能迭代
- 每个技能的 `version` 字段遵循 [SemVer](https://semver.org/) 规范
- 向后兼容的改进：minor 版本升级（1.0.0 -> 1.1.0）
- 破坏性变更：major 版本升级（1.x.x -> 2.0.0）
