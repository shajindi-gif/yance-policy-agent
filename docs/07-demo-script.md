## 演示步骤详解

### 步骤1：选择园区（30秒）

**操作动作**：
1. 打开 YanCe Policy Agent 工作台首页
2. 点击"选择园区"
3. 在园区列表中选择"徐汇AI创新园区"
4. 进入园区仪表盘

**展示内容**：
- 园区基础信息卡片（名称、地址、类型、入驻企业数）
- 园区政策服务概览（已录入政策数、已服务企业数、本月匹配报告数）
- 园区企业列表（显示前几家企业）

**讲解要点**：
> "我们先选择一个园区。这是徐汇AI创新园区，位于漕河泾开发区，有128家入驻企业，主导产业是人工智能。右侧可以看到园区的政策服务概览——系统已经录入了超过200条适用政策，本月已经生成了35份匹配报告。"

---

### 步骤2：录入企业（45秒）

**操作动作**：
1. 点击"新增企业"
2. 填写企业基础信息（使用默认示例数据）
3. 点击"构建画像"
4. 展示生成的企业画像

**展示内容**：
- 企业信息录入表单（快速填写）
- 企业画像报告（特征标签、行业分析、发展阶段评估）
- 画像完整度评分

**讲解要点**：
> "现在我们来录入一家新入驻的企业——衍策引擎，一家做 AI 政策服务的初创公司。我填写基础信息：行业是人工智能，细分领域是产业服务，成立于2024年，1到10人的初创团队。点击'构建画像'——系统自动生成了这家企业的多维画像，标注了'AI+产业服务'、'初创期'、'徐汇区注册'等特征标签。"

---

### 步骤3：匹配政策（60秒）

**操作动作**：
1. 在企业画像页面点击"匹配政策"
2. 系统展示匹配过程（加载动画）
3. 展示匹配结果列表
4. 展开一条政策的详细匹配报告

**展示内容**：
- 匹配结果列表（按匹配度排序，每条显示匹配度评分）
- 单条政策的详细匹配报告：
  - 政策名称和来源
  - 匹配依据（为什么这条政策适合这家企业）
  - 申请条件清单
  - 企业满足的条件和不满足的条件
  - 补贴范围说明
  - 申报截止日期
  - 政策原文链接

**讲解要点**：
> "现在点击'匹配政策'，系统会根据企业画像自动匹配适用政策。大家可以看到，系统找到了多条适用政策。我们展开第一条看看——系统不仅告诉你这项政策是什么，还详细列出了匹配依据：为什么这家企业适合申请、需要满足哪些条件、目前满足了哪些、还差哪些。每一项都有明确的政策原文引用，用户可以随时核验。请注意，系统不承诺申请成功，只是帮助企业了解信息。"

**重点强调**：
- 匹配结果可追溯（引用原文）
- 匹配逻辑可解释（列出匹配因素）
- 不承诺结果（标注免责声明）

---

### 步骤4：生成材料清单（45秒）

**操作动作**：
1. 在匹配结果页面选择2-3条高匹配度政策
2. 点击"生成材料清单"
3. 选择报告模板
4. 展示生成的材料清单和服务报告

**展示内容**：
- 申报材料清单（每项政策需要准备的材料列表）
- 政策服务报告（综合报告，包含匹配结果和材料清单）
- 报告预览（PDF格式）

**讲解要点**：
> "选择几条高匹配度的政策，点击'生成材料清单'。系统自动生成了每项政策需要的申报材料清单，以及一份综合性的政策服务报告。这份报告可以直接发送给企业，也可以由园区工作人员先复核再发送。报告包含园区品牌信息，看起来很专业。"

---

### 步骤5：生成服务工单（30秒）

**操作动作**：
1. 点击"创建服务工单"
2. 填写工单信息（自动关联企业和报告）
3. 指派给园区工作人员
4. 展示工单列表

**展示内容**：
- 服务工单创建表单
- 工单详情（关联企业、关联报告、指派人、截止时间）
- 工单列表（显示工单状态）

**讲解要点**：
> "最后，我们为这次服务创建一个工单。系统自动关联了刚才的企业和匹配报告，指派给园区的张经理。这样，整个政策服务过程就有了记录和追踪，不会遗漏。"

---

### 步骤6：输出园区服务报告（30秒）

**操作动作**：
1. 回到园区仪表盘
2. 点击"生成园区服务报告"
3. 展示生成的园区政策服务报告

**展示内容**：
- 园区政策服务报告（本月服务企业数、匹配报告数、工单完成率等）
- 数据可视化图表

**讲解要点**：
> "回到园区仪表盘，这里可以看到整个园区的政策服务概况。系统可以定期生成园区服务报告，帮助管理层了解服务成效。整个演示到这里——从选择园区到输出报告，不到5分钟，这就是 Policy Agent 的效率。"

## 演示注意事项

### 事前准备

- [ ] 确认Demo系统运行正常
- [ ] 确认示例数据已预置
- [ ] 确认网络连接稳定
- [ ] 准备备用方案（录屏视频）以防现场网络故障
- [ ] 测试投影/屏幕显示效果

### 演示技巧

- 操作时放慢速度，让观众能跟上
- 每个步骤之间做简短过渡，解释"下一步我们要做什么"
- 重点强调可追溯、可解释、不承诺的特点
- 如果某个步骤加载时间较长，用语言填充

### 应急方案

- 如果系统响应缓慢：提前录制一段操作视频作为备用
- 如果网络中断：使用本地缓存的Demo数据
- 如果某个功能异常：跳过该步骤，口头描述功能
- 如果时间不够：压缩步骤5和6，重点展示前4步

## 演示后常见即时提问准备

1. **"这些数据是真实的吗？"** → 演示数据是示例数据，政策来源是真实的政府公开信息，企业信息是虚拟的示例。
2. **"匹配准确率多少？"** → 诚实回答还在优化中，试点阶段会通过人工复核来验证和提升准确率。
3. **"这个演示用了多长时间？"** → 约4分钟，实际部署后园区工作人员的操作会更熟练。
