## 模块一：PolicyCopilot（政策问答Agent）

### 功能描述

PolicyCopilot 是一个基于 RAG（检索增强生成）架构的政策问答 Agent。用户可以通过自然语言向系统提问政策相关问题，系统会基于结构化政策知识库检索相关内容，并生成有据可依的回答。

### 输入

- 自然语言政策问题（如："AI初创企业可以申请哪些上海市级补贴？"）
- 可选的企业背景信息（行业、规模、所在地等）
- 多轮对话上下文

### 输出

- 结构化的政策回答，引用具体政策文件和条款
- 相关政策原文链接
- 建议的后续操作步骤
- 免责声明（"本回答仅供参考，不构成政策申报承诺"）

### 目标用户

| 用户角色 | 典型使用场景 |
|---------|------------|
| 园区工作人员 | 快速回答企业政策咨询，减少人工查询时间 |
| 入驻企业 | 自助查询政策信息，了解可享受的优惠政策 |
| 招商人员 | 在招商过程中快速展示园区政策优势 |

### 核心特性

- **可追溯**：每个回答都引用具体的政策文件，用户可以核验
- **时效性**：基于最新政策数据库，避免提供过时信息
- **上下文感知**：支持多轮对话，理解对话上下文
- **安全边界**：明确标注免责声明，不承诺申请结果
- **反馈机制**：支持用户反馈回答质量，持续优化

## 模块二：CompanyProfile（企业画像引擎）

### 功能描述

CompanyProfile 是企业的多维画像引擎。它通过收集和整合企业的多维信息，构建结构化的企业画像，为后续的政策匹配提供基础数据支撑。

### 输入

- 企业基础信息：名称、统一社会信用代码、注册地、成立时间
- 行业信息：所属行业、细分领域、主营业务
- 规模信息：员工人数、营收规模、融资阶段
- 创新信息：知识产权数量、研发投入、技术资质
- 资质信息：高新技术企业认定、专精特新认定、各类认证

### 输出

- 结构化企业画像报告
- 企业特征标签（如"AI+制造"、"初创期"、"高新技术企业"）
- 政策适用性初步评估
- 画像完整度评分（提示哪些信息还可以补充）

### 目标用户

| 用户角色 | 典型使用场景 |
|---------|------------|
| 园区工作人员 | 快速了解入驻企业情况，为服务提供依据 |
| 招商人员 | 分析目标企业特征，准备定制化招商方案 |
| 运营管理者 | 了解园区企业整体分布，优化运营策略 |

### 核心特性

- **渐进式画像**：从少量基础信息开始，随着使用逐步完善
- **最小必要原则**：不要求企业提供不必要的敏感数据
- **自动标签**：根据企业信息自动生成特征标签
- **隐私保护**：企业数据严格保密，支持数据删除
- **批量导入**：支持从Excel等格式批量导入企业信息

## 模块三：SubsidyMatch（政策匹配引擎）

### 功能描述

SubsidyMatch 是核心的政策匹配引擎。它基于企业画像和政策知识库，通过规则引擎和 AI 模型相结合的方式，将企业与适用政策进行精准匹配，生成详细的匹配报告。

### 输入

- 企业画像数据（来自 CompanyProfile 模块）
- 政策知识库（结构化的政策条目，含适用条件）
- 匹配偏好（如仅匹配国家级政策、仅匹配即将到期的政策等）

### 输出

- 政策匹配清单：按匹配度排序的适用政策列表
- 匹配报告：每条政策的匹配依据、申请条件、补贴范围、截止日期
- 匹配评分：每条政策的匹配度评分和关键匹配因素
- 缺口分析：企业目前不满足但可以通过努力达到的条件
- 行动建议：建议企业采取的下一步行动

### 目标用户

| 用户角色 | 典型使用场景 |
|---------|------------|
| 园区工作人员 | 为入驻企业生成政策匹配报告，提供精准服务 |
| 入驻企业 | 了解自身可享受的政策优惠，规划申报计划 |
| 招商人员 | 为目标企业展示入驻后可享受的政策优势 |

### 核心特性

- **精准匹配**：结合规则引擎和 AI 模型，提升匹配准确率
- **可解释**：每条匹配结果都附带详细的匹配逻辑和依据
- **可复核**：匹配结果支持园区工作人员人工复核
- **缺口分析**：识别企业不满足的条件，提供改进建议
- **动态更新**：政策库更新后自动重新匹配

## 模块四：BriefMaker（材料生成器）

### 功能描述

BriefMaker 是智能材料生成器。它基于政策匹配结果和企业信息，自动生成各类服务文档，包括申报材料清单、政策服务报告、企业政策简报等。

### 输入

- 政策匹配结果（来自 SubsidyMatch 模块）
- 企业画像数据（来自 CompanyProfile 模块）
- 模板选择（报告类型、格式偏好）
- 园区品牌信息（Logo、名称等，用于报告定制）

### 输出

- **申报材料清单**：针对每项政策，列出需要准备的申报材料
- **政策服务报告**：综合性的政策匹配报告，可用于正式场合
- **企业政策简报**：面向企业负责人的简明政策摘要
- **园区服务简报**：面向园区管理方的服务情况汇总
- **招商政策手册**：面向潜在入驻企业的园区政策优势介绍

### 目标用户

| 用户角色 | 典型使用场景 |
|---------|------------|
| 园区工作人员 | 快速生成标准化的服务文档，提升工作效率 |
| 园区管理者 | 阅读园区政策服务简报，了解服务运营情况 |
| 招商人员 | 使用招商政策手册展示园区优势 |

### 核心特性

- **模板化**：提供多种报告模板，一键生成
- **品牌定制**：支持园区品牌信息定制，提升专业感
- **格式灵活**：支持 PDF、Word、Markdown 等多种输出格式
- **内容可编辑**：生成的文档支持在线编辑和调整
- **版本管理**：历史报告可追溯，支持版本对比

## 模块五：ParkOps（园区运营工作台）

### 功能描述

ParkOps 是面向园区管理方和工作人员的运营工作台。它整合了上述四大模块的能力，提供了一个统一的管理界面，让园区工作人员可以在一个平台上完成政策服务的全流程管理。

### 输入

- 园区基础信息（名称、地址、类型、规模等）
- 入驻企业列表（可手动添加或批量导入）
- 工作人员账号和权限配置
- 服务流程和工单规则配置

### 输出

- 园区政策服务仪表盘（服务概况、关键指标）
- 企业政策服务列表（每家企业的匹配报告、服务记录）
- 服务工单管理（工单创建、分配、跟踪、完结）
- 数据报表和统计分析
- 服务商协同管理

### 目标用户

| 用户角色 | 典型使用场景 |
|---------|------------|
| 园区管理人员 | 查看园区政策服务整体情况，管理工作人员 |
| 园区服务人员 | 日常使用：管理企业、生成报告、处理工单 |
| 园区决策层 | 查看数据报表，评估政策服务成效 |

### 核心特性

- **统一管理**：一个平台管理所有企业和政策服务
- **工单系统**：将政策服务流程转化为可追踪的工单
- **数据看板**：实时查看园区政策服务关键指标
- **权限管理**：不同角色不同权限，保障数据安全
- **服务商协同**：管理与第三方服务商的合作关系
- **导出报告**：支持导出各类统计报表

## 模块协同流程

```
企业录入 → CompanyProfile（构建画像）
    ↓
PolicyCopilot（政策问答）← → SubsidyMatch（政策匹配）
    ↓                              ↓
BriefMaker（生成材料）← ← ← ← ← ← 
    ↓
ParkOps（创建工单 → 分配服务 → 跟踪进度 → 完结归档）
```

**典型工作流：**

1. 园区工作人员在 ParkOps 中录入或导入企业信息
2. CompanyProfile 自动构建企业画像
3. SubsidyMatch 基于画像匹配适用政策
4. BriefMaker 生成政策匹配报告和材料清单
5. 工作人员复核报告，创建服务工单
6. 跟踪服务进度，完结后归档

## 系统安全与合规

- **数据安全**：企业数据加密存储，严格访问控制
- **可追溯**：所有操作记录审计日志
- **不承诺**：系统不承诺政策申请成功或具体补贴金额
- **免责声明**：所有输出内容标注免责声明
- **数据删除**：企业可随时要求删除自身数据
- **不涉及金融**：不提供任何金融投资建议
